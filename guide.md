🔥 Projeto Inconformados - Guia de Vibe Coding (PWA)🎯 1. O "Reality Check" do Escopo (Tradução para Dev)Para entregar em 3 meses e dentro do orçamento, vamos enxugar as alucinações do cliente e focar na mecânica real:A "IA Inteligente": No MVP, não teremos um chatbot conversando com o jovem. A "IA" será passiva. No front-end, teremos mensagens motivacionais baseadas no Streak (se Streak = 7, exibe "Você está pegando fogo!"). A moderação do feed será uma chamada de API simples (OpenAI/Google) no backend (Edge Function no Supabase) apenas quando alguém postar algo.Modo Escola/Responsável: Fica para a Versão 2. O MVP é Jovem -> Líder (Admin).Sistema de XP: Todo hardcoded em constantes no front-end por enquanto, consumindo do JSON simulado.Design PWA Mobile-First: O layout inteiro será desenhado para telas de celular (max-width de 430px centralizado em telas grandes), com Bottom Navigation Bar.🚀 2. Os 10 Prompts para o Live ArtifactInstrução: Copie um prompt por vez e cole no OpenDesigner na aba "Live Artifact". Se algo sair errado, peça para ele corrigir na mesma conversa. Quando estiver perfeito, guarde o código.🛠️ Etapa 1: A Casca e o Design SystemPrompt 1: Layout Base PWA e Estilo"Crie o layout base (Shell) para o PWA 'Inconformados', focado 100% em Mobile First. Use React e Tailwind CSS. Defina uma paleta de cores jovem e moderna (Dark mode padrão com acentos em Verde Neon e Roxo). O layout deve ter uma área de conteúdo rolável e uma Bottom Navigation Bar fixa (com ícones do lucide-react: Home, Diário, Feed, Perfil). Use um container centralizado (max-w-md, mx-auto, h-screen) para simular a tela do celular no desktop. O conteúdo inicial deve ser apenas uma tela de boas-vindas."🔐 Etapa 2: Onboarding e LGPDPrompt 2: Cadastro e Login"Baseado no layout PWA anterior, crie a tela de Cadastro/Login. O visual deve ser muito limpo. Inclua campos de E-mail, Senha, Nome e Idade. Crucial: adicione um Checkbox obrigatório de 'Termos de Uso e LGPD' com um aviso amigável sobre privacidade. Use botões grandes e chamativos. Crie um objeto JSON simulado mockAuth para gerenciar os estados de loading e erro visualmente."🏠 Etapa 3: A Área do Usuário (O Coração do App)Prompt 3: Home Dashboard (Gamificação Principal)"Crie a tela inicial (Home) do aplicativo. Crie um objeto JSON mockUser no topo do arquivo com dados de: Nome, Nível atual (ex: '🟢 Construção'), XP (ex: 1200/2000), Streak (ex: 5 dias) e Notificações da 'IA'. A UI deve exibir:Saudação personalizada com avatar.Um Card de Nível com barra de progresso (XP).Um contador visual do Streak (foguinho).Uma lista de 'Missões de Hoje' (checkboxes interativos)."📖 Etapa 4: O Protocolo de EvoluçãoPrompt 4: Diário do Inconformado"Crie a tela 'Diário'. Essa tela é onde o jovem preenche sua rotina. Crie formulários bonitos usando cartões. Seções:'Como estou me sentindo?' (Seleção de Emojis).'Checklist de Disciplina' (Acordei no horário, Estudei, etc).'Reflexão' (Textarea para o diário em si).Adicione um botão de 'Salvar Diário' que simule um loading e depois mostre um aviso de '+50 XP ganhos'."📊 Etapa 5: O Perfil ProfundoPrompt 5: Perfil e Linha do Tempo"Crie a tela de 'Perfil do Usuário'. Crie um JSON mockProfileStats com dados de medalhas e histórico. A UI deve mostrar a foto, nome, núcleo/bairro e duas abas navegáveis na tela:Aba 1: 'Evolução' (Um gráfico simples ou barras de progresso mostrando Disciplina, Impacto Social, Liderança).Aba 2: 'Conquistas' (Grid com ícones representando medalhas conquistadas, ex: '15 Dias de Streak')."🌍 Etapa 6: A Rede Social SeguraPrompt 6: Feed Social Comunitário"Crie a tela 'Feed Social'. Crie um JSON mockFeed com uma lista de postagens (autor, foto, tempo, conteúdo de texto, selo de evolução, número de likes). A interface deve ter:Um campo minimalista no topo 'Compartilhe sua vitória hoje...'.A lista de posts em cartões, com botões para Curtir (ícone de coração) e Comentar. O design deve ser positivo e limpo, sem métricas tóxicas."🤝 Etapa 7: Sistema de ExpansãoPrompt 7: Tela 'Levante Um' (Convites)"Crie a tela 'Levante Um'. O objetivo é convidar amigos. A UI deve ter um visual de 'Missão Especial'.Um card em destaque com o 'Seu Código de Convite' e um botão de copiar.Uma lista de 'Meus Impactados' (JSON mockInvites), mostrando amigos que entraram pelo link e o nível atual deles.Um medidor mostrando quanto XP o usuário ganha pela evolução dos amigos."👑 Etapa 8: A Visão do Líder (Admin)Prompt 8: Painel Admin - Visão Geral"Crie uma tela de 'Painel do Líder'. Esta tela não usará o Bottom Navigation, será um layout diferente, focado em gestão. Use um JSON mockAdminStats. Mostre:Cards de visão geral (Total de Jovens, Jovens Ativos na Semana, Riscos de Evasão).Uma lista de 'Alertas da IA' (ex: 'João postou uma palavra bloqueada', 'Maria perdeu o streak de 30 dias').O visual deve ser de um painel de controle (Dashboard) gerencial limpo."👨‍🏫 Etapa 9: Gestão de Usuários (Admin)Prompt 9: Painel Admin - Lista de Jovens"Ainda no ambiente Admin, crie a tela de 'Lista de Membros'. Crie uma tabela ou lista em cards (pensando no mobile) que lista todos os jovens do núcleo. Inclua barra de pesquisa. Ao clicar num usuário (simulado), deve abrir um Modal/Drawer com botões de ação: 'Ver Evolução', 'Dar Medalha Manualmente', 'Ajustar XP'."✨ Etapa 10: O Refinamento Final PWAPrompt 10: Skeletons e Componentes de Feedback"Para polir o nosso PWA, crie uma tela de demonstração de 'Estados de Carregamento' (Loading States). Crie:Skeleton Loaders para o Feed e para os Cards da Home (aquelas caixas cinzas pulsantes de quando a internet está lenta).Um componente de 'Toast Notification' moderno para exibir mensagens flutuantes na tela (ex: 'Salvo sem internet, sincronizaremos depois' ou 'XP Adicionado!')."
